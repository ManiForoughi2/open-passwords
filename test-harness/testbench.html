Open Passwords - Test Bench

Load Open Passwords v0.6, make sure you have a saved login for openpw.test in the Passwords app, then work down the page. The black bar at the bottom logs every dropdown, fill, and input event as it happens.

1. Standard login dropdown SHOULD show + fill

Click the username field. You should see "Click to autofill". Click it - both fields fill (Touch ID prompt is expected), and the log shows input+change events.

2. Two-step login dropdown SHOULD show

Username alone (Google/Microsoft style). The dropdown should still appear because the field is autocomplete="username".

3. Search box dropdown should NOT show

A site search. Focus it - nothing should appear (this is the Instagram-tag class of bug).

4. Tag picker (Instagram repro) dropdown should NOT show

Formless combobox with placeholder "Search" - the exact field that used to wrongly trigger.

5. Comment box dropdown should NOT show

A comment field. No password on this card's form, so no login offer.

6. Newsletter email dropdown should NOT show

A lone email box, no password - should be treated as a newsletter, not a login.
