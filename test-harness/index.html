Password Manager Test Harness

Realistic mock pages for comparing two Chrome extensions side-by-side: Apple's official iCloud Passwords extension vs. an open-source alternative. Load both, walk each page, and record what each one does.

How to use this harness
  1. Serve this folder: ./serve.sh (or python3 -m http.server 8765), then open http://localhost:8765/.
  2. In Chrome, install/enable BOTH extensions. Test them ONE AT A TIME (enable A, run through all pages; then enable B, repeat) to avoid them interfering with each other.
  3. Use COMPARISON.md as your checklist and fill in what each extension does.
Standard login
login-standard.html

Classic username + current-password form on one view. The bread-and-butter autofill + save-prompt case.
Two-step login
login-twostep.html

Username first, password revealed by JS on a second view. Tests "username then password" detection on a dynamically shown field.
Sign-up
signup.html

new-password + confirm. Tests password generation and the save-new-credential prompt (and that it does NOT offer an existing login).
OTP - 6 separate boxes VIRAL BUG
otp-multibox.html

Six single-char maxlength="1" boxes. The pattern that makes Apple's extension nag with an "Enable AutoFill" balloon on every box. Does the extension show one prompt or six?
OTP - single field CONTROL
otp-singlefield.html

One one-time-code field - the correct pattern. The control to contrast against the multibox bug.
Forum FALSE-POSITIVE
forum.html

Real login in the header, plus a search box, reply box, topic field, and newsletter email that are NOT logins. Does the extension wrongly offer to fill them?
Tricky edge cases EDGE
tricky.html

Four hard cases on one page: a login inside an iframe, a hidden password field, a password field with no name/id, and a login form injected via setTimeout after 1.5s (tests MutationObserver re-scanning).
